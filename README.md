## Структура проекта

| Путь | Описание |
|------|----------|
| `configuration/startup_db` | Стартовая конфигурация r1 и r2 |
| `script/prepare-ssh` | Автоматическое прокидывание SSH-ключей |
| `gateway/gateway.py` | Код шлюза |
| `docker-compose.yml` | Описание инфраструктуры |
| `Dockerfile` | Сборка контроллера из исходников на GitHub |

### Хранилище конфигураций (`storage/`)

По умолчанию хранит только итоговый `openconfig.json`. Остальные файлы - это примеры промежуточных состояний.

| Файл | Описание |
|------|----------|
| `openconfig` | Итоговый JSON |
| `raw_openconfig.json` | JSON после преобразования, с дублирующимися `xmlns` |
| `openconfig.xml` | Исходная XML-конфигурация |

---

## Запуск

```bash
# 1. Поднять контейнеры
docker compose up -d

# 2. Зайти в контейнер контроллера
docker exec -it controller bash

# 3. Запустить шлюз
python3 /usr/local/bin/gateway.py
```

---

## Использование

Шлюз предлагает три действия:

### 1. Получить конфигурацию устройства
Выбрать пункт **1**, затем нужное устройство. Конфигурация сохранится в `storage/`.

### 2. Просмотреть конфигурацию
Выбрать пункт **2**, затем нужное устройство. Конфигурация выведется в терминал.

### 3. Применить изменения
Отредактировать нужный `openconfig` в `storage/`, затем выбрать пункт **3**.
